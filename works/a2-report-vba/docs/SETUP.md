# ReportHub 導入手順(所要 約10分)

対象: Windows版 Excel 2016以降(32bit/64bit両対応)

## 1. ブックの準備

1. 新規マクロ有効ブック(.xlsm)を作成する(このブックが集計側。報告ブックには一切手を入れない)
2. `設定` シートを作成し、A列にキー・B列に値:

| A列 | B列(例) |
|---|---|
| 報告書フォルダ | C:\Reports\2026-06(または `sample-data\reports` の絶対パス) |

残りのシート(統合データ・ダッシュボード・取込結果・エラーログ)は初回実行時に自動作成される。

## 2. マクロの取り込み

Alt+F11 → ファイル→インポートで `dist/cp932/` の4つの `.bas` を取り込む(src/はUTF-8正本。日本語Windowsでは dist/cp932/ を使う)。

## 3. 実行

Alt+F8 → `RefreshDashboard`。完了ダイアログに取込ファイル数・統合行数・処理時間が表示される。

- `ダッシュボード` シート: 月次推移(前年比)・担当者ランキング・商品TOP5
- `取込結果` シート: ファイルごとの成功/警告/失敗(色分き)。サンプルには警告1冊が意図的に含まれている

## 4. 報告ブック側の前提(ゆるい)

- 1シート目に明細表があること
- 1〜10行目のどこかに「日付/担当者/商品/売上」系のヘッダーがあること(別名・列順・前置行は自動吸収)
- これに合わない様式はカスタマイズで対応(ヘッダー別名の追加は modScan の1箇所)

## 5. トラブルシューティング

| 症状 | 対処 |
|---|---|
| 「報告書フォルダが見つかりません」 | 設定のパスが実在するか。ネットワークドライブは接続状態を確認 |
| 特定ファイルが「警告」になる | ヘッダー名が対応表にない → modScan.FindHeader の Select Case に別名を1行追加 |
| 数字が合わない | 取込結果の「除外行」を確認(小計行は自動除外される仕様) |
