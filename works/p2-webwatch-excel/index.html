WebWatch — 定型Web情報収集 → 差分検知 → Excelレポート(Python)

誰の・どんな困りごとを解決するか

毎朝、複数サイトの公開情報(公的統計の新着・自社取扱商品の公開価格・入札公告・新着一覧)を目視でチェックして表に転記している担当者。「昨日から何が変わったか」だけが知りたいのに、全ページを見て回っている。

Before / After

Before(手作業) After(WebWatch)
毎朝の巡回 30分×毎日 コマンド1発(タスクスケジューラで全自動化も可)
変化の見落とし 目視のため頻発 前回スナップショットとの差分を自動検知
記録 手でExcelに転記 色分き整形済みExcelが自動生成(新規=緑・消滅=赤・オートフィルタつき)

設計方針: 「行儀の良い収集」を実装レベルで強制

  • robots.txt を確認し、拒否されたURLは取得しない(コードが拒否する)
  • 同一ドメインへのアクセス間隔制御(設定で1秒未満にはできない)
  • User-Agent と連絡先を明示
  • 規約・法令に反する収集(ログイン突破・大量アクセス・禁止サイト)は請け負いません — この方針自体が、依頼者を守る品質です

機能一覧

  • YAML設定だけで対象追加(URL・1件分のCSSセレクタ・列名→相対セレクタ)。コード変更不要
  • 文字コード自動判定・タイムアウト/接続失敗時も1対象の失敗で全体を止めない(失敗は構造化ログに記録し終了コードで通知)
  • 差分検知: 新規行(NEW・緑)/消滅行(赤・末尾に記録=黙って消さない)
  • 差分サマリシート(対象×件数×変化の一覧)+対象別シート(ヘッダー固定・オートフィルタ)
  • 構造化ログ(JSON 1行)— 定時実行の監視と相性が良い

動かし方

▶ かんたん起動(コマンド不要): 解凍したフォルダの中の - Mac: run_mac.command をダブルクリック - Windows: run_windows.bat をダブルクリック

初回のみ環境構築を自動で行い、config.yaml を用意して収集→Excel出力まで実行します(初回は雛形を作って一旦停止するので、対象サイトを記入して再実行)。Python未導入なら案内だけを表示します。

実案件では定期実行まで自動化: 「毎朝9時に自動収集して結果だけメール」など、クライアントが何も操作しない形(スケジューラ登録・こちらでのホスト運用)でお渡しできます(DeliveryStandards.md 参照)。

開発者向け(手動):

pip install -r requirements.txt
cp config.example.yaml config.yaml
python -m webwatch config.yaml
pytest tests/                        # テスト12件(抽出・差分・Excel・設定検証・失敗系)

実案件ではこう応用できます

  1. 競合価格・在庫の定点観測(規約確認のうえ)→ 変化があった朝だけメール通知
  2. 入札公告・助成金の新着監視 → キーワード一致のみ抽出してSlackへ
  3. 自社サイトの掲載チェック(全店舗ページの電話番号・営業時間の一括検証)

※ 同梱の設定例はダミーURLです。実対象は利用規約の確認後に設定します。
