# FormFlow — Googleフォーム受付自動化(GAS)

## 誰の・どんな困りごとを解決するか

イベント・教室・予約の申込みをGoogleフォームで受けている主催者。**申込みが来るたびに、確認メールを手で送り、台帳へ転記し、定員を数えている**。夜中の申込みは翌朝まで放置になり、二重申込みにも気づけない。

## Before / After

| | Before(手作業) | After(FormFlow) |
|---|---|---|
| 確認メール | 気づいたら手動送信(深夜は翌朝) | **送信と同時に自動返信(24時間)** |
| 台帳転記 | フォーム回答からコピペ | 自動転記+重複申込を色付き検知 |
| 定員管理 | 都度数える。超過受付の事故 | 定員到達で**フォーム自動クローズ** |
| 状況把握 | シートを開いて確認 | 毎日18時に申込サマリが届く |

## 機能一覧

- フォーム送信トリガーで確認メール自動返信(テンプレートに `{{名前}}` 等を差し込み)
- 台帳シートへの整形転記・同一メールアドレスの重複検知(行を黄色でマーク)
- 管理者への即時通知(メール+Slack等のWebhook対応)
- 定員到達でフォームを自動で受付終了(締切メッセージも自動設定)
- 日次サマリの自動送信(本日n件/累計n件/定員残り)
- **多重発火対策(ロック)・エラー時の管理者即時通知・実行ログシート** — 「動いているつもりで止まっていた」を防ぐ
- 設定はすべて「設定」シート(コード変更不要)。テスト用関数つき

## 構成・動作環境

```
src/Code.gs   スクリプト本体(1ファイル・貼り付けるだけ)
docs/SETUP.md 導入手順(20分・スクリーンショット手順)
```

Googleアカウントのみで動作。**追加費用ゼロ**(Gmail送信上限: 無料100通/日)。

## 実案件ではこう応用できます

1. **抽選制イベント**: 受付→抽選→当選/落選メールの一括送信
2. **予約枠管理**: 日時ごとの枠数管理と満枠の選択肢自動非表示
3. **社内申請フロー**: 申請→上長へ承認依頼→承認で本人へ通知(稟議の簡易電子化)
