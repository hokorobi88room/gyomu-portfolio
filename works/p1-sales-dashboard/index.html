SalesLens — 売上データ分析ダッシュボード(Python / Streamlit)

誰の・どんな困りごとを解決するか

売上データがExcelで開くのがつらいサイズ(数万〜数十万行)になってきた事業者。ピボットテーブルで毎月同じ集計を繰り返し、前年比の計算とグラフ作りに半日かけている。「Excel実務6年だからこそ、Excelでやるべきことと、Pythonに移すべきことの境目がわかる」を体現する作品。

Before / After

Before(Excel手作業) After(SalesLens)
月次レポート作成 ピボット+グラフで半日 ファイルを放り込んで10秒
前年比 都度数式を組む 自動計算・グラフ表示
異常な日の発見 目視(見逃す) ±3σの統計的検知(感度調整可)
列名がバラバラのデータ 手で直してから読み込み 売上/金額/売上高 等を自動吸収

機能一覧

  • CSV(UTF-8/Shift_JIS自動判定)/Excel のドラッグ&ドロップ読み込み
  • 列名ゆらぎの自動吸収(日付/売上日/取引日、売上/金額/売上高、商品/品目 等)
  • クレンジング: 不正行を理由別に計数して報告(黙って捨てない)
  • KPIカード/月次推移+前年同月比の複合グラフ/商品別TOP10/地域別構成(plotlyでインタラクティブ)
  • 異常日検知(日次売上の±σ検知・スライダーで感度調整・標本不足時は誤検知防止で停止)
  • Excelレポート出力(月次・商品別・地域別・整形済みデータの4シート)— 分析結果をExcel文化の職場へ持ち帰れる

動かし方

▶ かんたん起動(コマンド不要): 解凍したフォルダの中の - Mac: run_mac.command をダブルクリック - Windows: run_windows.bat をダブルクリック

初回のみ環境構築を自動で行い(1〜2分)、ブラウザでダッシュボードが開きます。Python未導入なら案内と入手先だけを表示します(macOS初回は右クリック→「開く」で警告を解除)。

実案件での納品は、これをさらに手間ゼロにします: Streamlit Community Cloud(無料)へホストし、URLを開くだけの形でお渡しできます(DeliveryStandards.md 参照)。

開発者向け(手動):

pip install -r requirements.txt
streamlit run app.py
pytest tests/               # テスト11件(正常系・異常系・10万行3秒以内の性能)

品質

  • UI(app.py)とロジック(core.py)を分離し、コアはpandas純関数としてテスト11件で担保(型ヒント・ruff準拠)
  • 性能テスト: 10万行の全分析パイプラインを3秒以内で完走することをテストで保証
  • 実バグをテストが検出→修正済み(商品名が空欄の行の除外漏れ)— テストを書く開発姿勢ごと見せる

実案件ではこう応用できます

  1. 定期自動レポート: 毎朝データを取り込みSlack/メールへサマリ送信(ダッシュボードを見に行かなくていい)
  2. 複数店舗・複数事業の統合分析(A2のReportHubで集めたデータの出口)
  3. 在庫・勤怠・広告データへの横展開(コアの集計ロジックを差し替えるだけ)

※ サンプルデータは架空の数値です。
